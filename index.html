Privacy Policy

Extract Text from Image · Last updated: July 19, 2026

TL;DR: All OCR processing happens locally on your device — your images, screenshots and recognized text never leave your browser. The extension sends only minimal anonymous usage statistics (a random ID, extension version and interface language) so we know how many people use it.

What data we collect

We do not collect any personally identifiable information, browsing history, page content, screenshots, images, or recognized text.

We collect anonymous usage statistics: when the extension is installed, used (one "active today" ping per day) or uninstalled, it sends a small event to our server containing a random identifier (a UUID generated inside the extension, not linked to you or your Google account), the extension version, and your browser interface language. Our server does not store IP addresses or user-agent strings.

Optional feedback

If you choose to rate the extension and leave a comment, or select an uninstall reason on the page shown after removal, the text you enter is sent to our server together with the same random identifier and extension version. This is entirely optional and contains no email address or account information.

How OCR works

When you click the extension icon and select a region of a webpage, the extension captures a screenshot of that region using Chrome's built-in API. The screenshot is processed entirely within your browser using a local PaddleOCR machine learning model (WebAssembly). The recognized text appears in a panel inside your browser tab. No image, text, or metadata is sent to any external server.

Model files

On first use, the extension downloads OCR model files (~15 MB total) from a public GitHub repository:

  • github.com/PT-Perkasa-Pilar-Utama/ppu-paddle-ocr-models

This is a standard HTTPS download, identical to loading any public webpage. GitHub may log the request as part of standard server operations. After download, model files are cached locally — no further network requests are made for OCR processing.

Settings storage

The extension stores your selected language preference using Chrome's storage.local API. This data stays on your device and is never transmitted anywhere.

Permissions explained

  • activeTab — to capture a screenshot of the page region you select
  • scripting — to inject the selection UI into the current page when you click the icon
  • offscreen — required to run the OCR WebAssembly engine in a sandboxed background context
  • storage — to remember your language preference between sessions
  • alarms — to schedule the once-a-day anonymous "active today" ping
  • host_permissions (all_urls) — to allow text extraction from any webpage you choose to use the extension on. The extension only activates when you explicitly click the icon

Third-party services

The extension does not use any third-party analytics, advertising networks, or tracking pixels. Anonymous usage statistics and optional feedback are sent to our own self-hosted server (devexthub.com) and are never shared with or sold to anyone. The only other external request is the one-time download of OCR models from GitHub (described above).

Data deletion

To remove all data associated with this extension, simply uninstall it from Chrome. This clears all stored preferences and cached model files. Because usage statistics are tied only to a random identifier, they cannot be linked back to you; if you want the events for your random ID deleted from our server, contact us via the Chrome Web Store listing.

Changes to this policy

If we change this policy in the future, the "Last updated" date above will be modified. Material changes will be communicated via the Chrome Web Store listing.

Contact

Questions about privacy? Open an issue on the project's GitHub repository or contact the developer through the Chrome Web Store listing.
